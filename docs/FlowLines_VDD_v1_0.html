Section 01
Colour System

Two colour roles operate in parallel: the UI palette (background scale, brand purple, gold accent) and the game path colours (8 vibrant hues, one new colour introduced each pack). All path colours must remain visually distinct against deep purple at small cell sizes and render acceptably for deuteranopia/protanopia through shape + brightness differentiation.

Section 06
Animation Specification
Path draw
Continuous · on drag
Coloured line extends cell-by-cell as finger drags. Immediate — no delay or easing. Path colour fills cell background at 22% opacity on enter. Round linecap + linejoin for organic feel.
Path lock-in
200ms ease-out · on dot reached
When a path reaches its matching endpoint, the full path does a single shimmer pass from source to target. Dot endpoint ring pulses: glow radius 0 → 12px → 6px. Duration 200ms ease-out.
Win — board fill
600ms total · on 100% coverage
All cells flash white simultaneously, then each cell's colour brightens from 22% → 85% opacity in a radial cascade outward from board centre. Followed immediately by the star rating animation.
Star rating
120ms × 3 · 130ms stagger
Stars bounce-scale into view one at a time: scale(0) → scale(1.3) → scale(1.0). 120ms per star, 130ms stagger between stars. Gold glow on each star on landing. 3-star gets additional sparkle particles around the result panel.
Hint reveal
800ms repeat · on ad complete
Optimal next cell pulses: white border 1px → 3px, cell background flashes white at 30% opacity, repeating every 800ms until the correct cell is tapped. Colour-agnostic — always white regardless of path colour.
Pack unlock
500ms total · on gate cleared
Locked pack card: shake-rattle → gold border sweeps in from left → lock icon dissolves to checkmark. "Pack Unlocked!" toast slides up from bottom, auto-dismisses after 2 seconds.
Undo
150ms ease-in · on tap
Path retracts one cell backward with a fast fade (150ms ease-in). Cell reverts to empty state. Move count decrements with a brief number flash. No undo animation for completed/locked paths.
Screen transitions
200ms slide
React navigation: new screen slides in from right (200ms ease-in-out). Back navigation slides from left. Result screen fades up from bottom over the game screen.
Ambient background
4s loop · continuous
Radial purple glow at screen centre slowly breathes (opacity 0.07 → 0.12, 4s ease-in-out loop). Dot grid texture is fixed — no scroll. No floating particle canvas (unlike Numtap) — kept clean for puzzle focus.
No rescue flash. Flow Lines has no rescue pill mechanic (unlike Numtap). The hint system is the only ad-triggered animation during gameplay. The GET A CLUE amber tile flash and LOW ON TIME green pulse do not exist in this game.

Phaser GameScene Rendering Rules
Cell size calculation
cellSize = (screenWidth − padding×2 − gap×(N−1)) / N. Use padding=8px each side. Gap=3px for 6×6/7×7, 2px for 8×8/9×9. Minimum cellSize = 28px.
Dot endpoint rendering
Phaser Graphics circle at 70% of cellSize. Outer glow ring: same colour at 35% opacity, radius +4px. Both at cell centre. No shadow filter — use rgba glow only.
Path stroke
Phaser Graphics.strokePath() between cell centres. lineWidth = 40% of cellSize. lineCap = 'round'. lineJoin = 'round'. Colour at 100% opacity for stroke, 22% opacity rect for cell fill.
Grid background
Phaser RenderTexture for grid base. Cells: rounded rects (rx=3) filled with bgDeep + 3% white. Grid lines: purple at 12% opacity. All flat — no box shadows for performance.
Timer component
React only (not Phaser), same pattern as Numtap. Only rendered in Timed Mode. Classic, Daily, and Zen modes: no timer DOM element at all.
Coverage display
CoverageCalc.ts emits % to gameStore on every cell change. React reads and renders both the text HUD value and the gradient progress bar. No Phaser canvas element for coverage.
Reminder — GameScene.ts is locked after Sprint 2 sign-off. Same rule as Numtap. All Phaser rendering logic must be finalised and approved before any UI or service work begins in Sprint 3.
